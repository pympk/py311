# Summary: It Ain't Broke (AI Engineer Europe)
**Speaker:** Matt Pocock  
**Theme:** Why software engineering fundamentals are more critical than ever in the age of AI-assisted coding.

## 1. The "Specs to Code" Illusion
Many believe we’ve entered a new paradigm where we simply provide specifications and AI generates the code. However, relying solely on this often leads to:
- **Software Entropy:** Code that starts okay but becomes "garbage" after several iterations.
- **Complexity:** Systems that become increasingly hard to understand and modify.
- **The Cost of Bad Code:** Contrary to the belief that "code is cheap" now, bad code is actually more expensive because it hinders the AI’s ability to help you effectively in the future.

---

## 2. AI Failure Modes & Strategic Tips

### Failure Mode #1: The AI didn't do what I want.
*   **The Issue:** A lack of shared design concept. No one knows exactly what they want at the start.
*   **Tip:** Reach a shared design concept before coding.
*   **Skill:** `/grill-me` – Instruct the AI to interview you relentlessly until a shared understanding is reached.

### Failure Mode #2: The AI is way too verbose.
*   **The Issue:** A language gap between the developer and the "domain expert" (the AI).
*   **Tip:** Use **Ubiquitous Language** (from Domain-Driven Design).
*   **Skill:** `/ubiquitous-language` – Create a `.md` file defining terms and concepts that both you and the AI use consistently.

### Failure Mode #3: The code doesn't work.
*   **The Issue:** Lack of validation.
*   **Tip:** Implement tight feedback loops:
    *   Static typing (TypeScript).
    *   Browser access (for front-end).
    *   Automated tests.

### Failure Mode #4: The AI is doing way too much.
*   **The Issue:** "Outrunning your headlights." Taking steps too large to verify.
*   **Tip:** **Test-Driven Development (TDD).** The rate of feedback is your speed limit.
*   **Skill:** `/tdd` – Force the AI to write a test first, make it pass, then refactor.

### Failure Mode #5: The AI doesn't understand my code.
*   **The Issue:** "Shallow Modules" (too many tiny, complexly interfaced pieces).
*   **Tip:** Aim for **Deep Modules** (lots of functionality behind a simple interface).
*   **Skill:** `/improve-codebase-architecture` – Periodically ask the AI to refactor for better module "depth."

### Failure Mode #6: "My Brain Hurts" (High cognitive load).
*   **The Issue:** Trying to review every line of AI-generated implementation.
*   **Tip:** Design the interface; delegate the implementation.
*   **Concept:** Treat deep modules as "Grey Boxes." If the interface is solid and the tests pass, you don't always need to internalize every detail of the implementation.

## 3. The Core Message
AI is a powerful **tactical** programmer (a "sergeant on the ground"). To use it effectively, you must be a **strategic** programmer. This requires a deep investment in software fundamentals:
- Understanding complexity.
- Managing entropy.
- Designing robust architectures.

**"Bad code is the most expensive it's ever been."**

✦ The video emphasizes that as AI makes coding faster, software engineering fundamentals become even more critical
  to prevent "AI-generated spaghetti code." Here are the key software principles discussed:

  1. Ubiquitous Language (Domain-Driven Design)
   * Concept: Using a shared, consistent vocabulary between developers, stakeholders, and now, AI agents.
   * Importance: AI is highly sensitive to naming. Clear, descriptive names for classes, methods, and variables
     help the AI understand the context and purpose of the code, leading to better suggestions and fewer bugs.

  2. Vertical Slices
   * Concept: Building and delivering functional, end-to-end features (from UI to database) rather than working in
     horizontal layers (all UI, then all API, then all DB).
   * Importance: This approach allows for faster feedback loops and ensures that each increment provides actual
     value. AI tools excel when tasked with implementing specific, cohesive features.

  3. Test-Driven Development (TDD)
   * Concept: Writing tests before writing the functional code.
   * Importance: In an AI-assisted workflow, tests act as the "spec" and the "safety net." TDD ensures that the
     code generated by the AI actually does what it's supposed to do and prevents regressions as the codebase
     grows.

  4. Deep Modules (A Philosophy of Software Design)
   * Concept: Creating modules that provide powerful functionality through a simple, narrow interface (hiding
     complexity).
   * Importance: This minimizes the "cognitive load" for both humans and AI. By keeping interfaces simple, you
     reduce the surface area where the AI can make mistakes or create unnecessary dependencies.

  5. Iterative Refinement
   * Concept: Treating software development as a continuous process of building, testing, and refactoring.
   * Importance: AI is an assistant, not a replacement for architectural oversight. These principles provide the
     structure needed to manage the high volume of code AI can produce, ensuring the system remains maintainable
     and scalable.
